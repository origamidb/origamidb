# Notes

Scratch space for working drafts, design exploration, mining notes from external research, and anything that wants to live in the repo before it earns a place in the curated layer.

Material here is not authoritative. Nothing in `notes/` commits the project to anything; do not cite from `notes/` as if it were specification. Decisions land in [`../adr/`](../adr/), substantive proposals in [`../../rfcs/`](../../rfcs/), authoritative descriptions in [`../spec/`](../spec/).

Low ceremony. Date-prefixed filenames (`YYYY-MM-DD-short-title.md`) are conventional but not required. PR review focuses on whether the draft is self-contained and readable; exhaustiveness, polish, and consensus are explicitly not the bar.

When a draft makes claims that aren't grounded in stated project sources ([`README`](../../README.md), [`ARCHITECTURE.md`](../../ARCHITECTURE.md), accepted ADRs and RFCs, spec pages), mark them as inferences — `*inferred:*` or `[unsettled]`. The whole point of this folder is exploration, but exploration that's labeled.
